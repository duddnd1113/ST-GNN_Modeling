# Top-k Multi-View Attention CluTSP prototype

This repository adapts your previous **top-k POMO TSP** snapshot workflow into a **Clustered TSP (CluTSP)** workflow using a **multi-view attention encoder-decoder**.

```text
full scored grid -> select top-k high-priority cells -> cluster selected cells -> solve CluTSP
```

Compared with the previous TSP version, this version forces the route to visit nodes in the same cluster consecutively before moving to another cluster.

## Main files

- `CluTSPProblemDef.py`  
  Top-k selection, 8-view coordinate augmentation, KMeans cluster assignment, and graph construction.

- `Sublayer.py`  
  Scaled dot-product attention and multi-head attention used by the decoder.

- `MVAttnCluTSPModel.py`  
  Multi-view dense GATv2-style encoder + collaborative decoder.
  - Global graph view: full graph with same-cluster edge attributes.
  - Local graph view: intra-cluster graph only.
  - Global Guiding Module: chooses cluster-level direction.
  - Local Routing Module: chooses the next node.

- `train_test_visualize_snapshot_clutsp.py`  
  One-snapshot demo matching your existing `train_test_visualize_snapshot.py` style.

## Install

This version is self-contained PyTorch and does **not** require `torch_geometric`. Python 3.8+ is recommended.

```bash
pip install -r requirements.txt
```

## Run one static demo

```bash
python train_test_visualize_snapshot_clutsp.py \
  --total-grid-size 22500 \
  --top-k 50 \
  --n-cluster 10 \
  --epochs 300 \
  --multi-start 50 \
  --output-route clutsp_route_output.csv \
  --output-fig clutsp_route_visualization.png
```

For a faster CPU sanity check:

```bash
python train_test_visualize_snapshot_clutsp.py \
  --total-grid-size 2500 \
  --top-k 30 \
  --n-cluster 6 \
  --epochs 5 \
  --multi-start 8
```

## How this differs from your previous TSP snapshot

Previous version:

```text
select top-k -> solve ordinary TSP over selected nodes
```

This version:

```text
select top-k -> cluster selected nodes -> solve CluTSP with cluster-contiguity constraint
```

So the route is less free than ordinary TSP, but more operationally realistic if each cluster represents a management zone, road-cleaning district, administrative unit, or garage-centered service region.

## Notes

- The depot is set to `(0.5, 0.5)` by default. Change with `--depot-x`, `--depot-y`.
- Top-k nodes are selected by `score`.
- Clusters are generated by KMeans over selected top-k nodes.
- The route CSV includes `route_order` and `cluster` columns.
